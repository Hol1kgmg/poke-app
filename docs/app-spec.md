# ポケモン相性診断アプリ 仕様書

## 1. アプリ概要

| 項目 | 内容 |
|---|---|
| アプリ名 | ポケモン相性診断 |
| コンセプト | 2匹のポケモンの「運命の相性」を占い風に診断するアプリ |
| 表示形式 | 相性スコアを％で表示 |
| 特徴 | 計算ロジックをユーザーに把握されない設計（BFF構成） |

---

## 2. システム構成

```
[フロントエンド]  ←→  [BFF]  ←→  [PokeAPI]
```

| レイヤー | 役割 |
|---|---|
| フロントエンド | ID入力・結果表示のみ |
| BFF | PokeAPI呼び出し・フィールド抽出・相性計算 |
| PokeAPI | ポケモンデータの提供（外部API） |

---

## 3. リクエストフロー

```
【フロントエンド】                【BFF】                    【PokeAPI】
      |                             |                            |
      | ① ID×2を送信               |                            |
      |  { id_a: 1, id_b: 25 }     |                            |
      |────────────────────────────>|                            |
      |                             | ② pokemon/{id_a} をリクエスト
      |                             |─────────────────────────────>|
      |                             | ③ ポケモンAの全データ        |
      |                             |<─────────────────────────────|
      |                             | ④ pokemon/{id_b} をリクエスト
      |                             |─────────────────────────────>|
      |                             | ⑤ ポケモンBの全データ        |
      |                             |<─────────────────────────────|
      |                             |                            |
      |                             | ⑥ 必要フィールドを抽出      |
      |                             |   + SHA-256で相性計算       |
      |                             |                            |
      | ⑦ 計算結果のみ返却          |                            |
      |  { score: 73,               |                            |
      |    name_a: "フシギダネ",    |                            |
      |    name_b: "ピカチュウ",    |                            |
      |    img_a: "...",            |                            |
      |    img_b: "..." }           |                            |
      |<────────────────────────────|                            |
      |                             |                            |
      | ⑧ 結果を表示               |                            |
```

## 4. API仕様

### 4-1. フロントエンド → BFF

**エンドポイント**
```
POST /api/match
```

**リクエストボディ**
```json
{
  "id_a": 1,
  "id_b": 25
}
```

| フィールド | 型 | 説明 |
|---|---|---|
| id_a | number | ポケモンAの図鑑ID |
| id_b | number | ポケモンBの図鑑ID |

**レスポンスボディ**
```json
{
  "score": 73,
  "name_a": "フシギダネ",
  "name_b": "ピカチュウ",
  "img_a": "https://raw.githubusercontent.com/.../1.png",
  "img_b": "https://raw.githubusercontent.com/.../25.png"
}
```

| フィールド | 型 | 説明 |
|---|---|---|
| score | number | 相性スコア（0〜100） |
| name_a | string | ポケモンAの名前 |
| name_b | string | ポケモンBの名前 |
| img_a | string | ポケモンAの画像URL |
| img_b | string | ポケモンBの画像URL |

> ⚠️ 計算に使用したフィールドや中間データは一切返却しない

---

### 4-2. BFF → PokeAPI

**使用エンドポイント**
```
GET https://pokeapi.co/api/v2/pokemon/{id}
GET https://pokeapi.co/api/v2/pokemon-species/{id}
```

| エンドポイント | 取得目的 |
|---|---|
| /pokemon/{id} | タイプ・アビリティ・画像URL・名前など |
| /pokemon-species/{id} | 生息地・タマゴグループ・色・体型・世代など |

## 5. 相性計算ロジック（BFF内部）

> ⚠️ このセクションはBFF内部の実装詳細。フロントエンドには非公開。

### 計算方式：SHA-256ハッシュ

**詳細：保留**（別途決定）

**方針**
- 各ポケモンから抽出したフィールドを文字列に変換して結合
- 結合した文字列をSHA-256でハッシュ化
- ハッシュ値の先頭バイトを0〜100にマッピングしてスコアとする
- 同じID組み合わせは常に同じスコアを返す（再現性あり）
- AB と BA は同じ結果になるよう順番を正規化する（id の小さい方を先にする等）

## 6. フロントエンド仕様

### 画面構成（シンプル版）

```
┌─────────────────────────────┐
│  ポケモン相性診断            │
│                             │
│  [ID入力A]   [ID入力B]      │
│                             │
│      [ 診断する ボタン ]     │
│                             │
│  ┌─────────┐  ┌─────────┐  │
│  │ 画像A   │  │ 画像B   │  │
│  │ 名前A   │  │ 名前B   │  │
│  └─────────┘  └─────────┘  │
│                             │
│       ❤️ 相性: 73%          │
└─────────────────────────────┘
```

### インタラクション

| アクション | 挙動 |
|---|---|
| ボタン押下 | BFFへリクエスト送信・ローディング表示 |
| 結果取得 | ポケモン画像・名前・スコアを表示 |
| エラー時 | IDが存在しない場合はエラーメッセージ表示 |
